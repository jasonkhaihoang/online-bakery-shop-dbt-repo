# Processing Business Intent to dbt Artefacts

**Type:** Master Orchestration Skill
**Phase:** Entry Point
**Depends on:** `project_standards.md`

## Purpose

Master skill that analyzes incoming business intents and orchestrates the correct sequence of sub-skills to generate dbt models, data contracts, tests, and artefacts. **Generates all models and tests**, then **executes tests to ensure models are logically correct** before completion.

## When to Use

Invoke this skill when a new business intent arrives (sales report, customer analysis, dashboard requirement, metric definition, etc.). The skill will:
1. Analyze the business intent against project standards
2. Determine which sub-skills are needed
3. Orchestrate generation in the correct sequence (modeling, then test generation)
4. Execute unified `dbt test` to validate all tests pass
5. Ensure all required artefacts are generated

## Input Requirements

1. **Clear business intent description** — e.g., sales report, customer cohort analysis, revenue dashboard
2. **Business logics, metrics, and rules** —
   - Key business metrics required (e.g., total revenue, customer lifetime value, order count)
   - Key business rules each model must enforce (e.g., revenue > 0, status in allowed set, no duplicates, null handling strategy)
3. **Grain of the primary output** — e.g., one row per order, one row per customer per month, one row per product per day
4. **Known edge cases and data conditions** — e.g., null handling, historical changes, SCD behavior, data quality issues

**Accept:** Text input or uploaded `.md` files. If anything is missing, prompt for input.

## Execution Flow

**Always generate for any intent:**
1. `1.1` — generating-dbt-mart-models-and-data-contracts
2. `1.2` — generating-dbt-staging-int-models-and-data-contracts
3. `1.4` — generating-dq-checks (generate only; not executed in Phase 2)
4. `2.1` — generating-dbt-unit-tests (generate only)
5. `2.2` — generating-dbt-integration-tests (generate only)
6. `2.3` — generating-dbt-e2e-tests (generate only)

**Then execute:**
7. `dbt test` — Execute all generated tests (2.1 unit + 2.2 integration + 2.3 E2E) and report results

**Conditionally generate:**
- `1.3` — generating-dbt-semantic-models (optional, if using Semantic Layer)

**Post-multiple-intents (separate phase):**
- Phase 3: refactoring-dbt-project (run after 3+ business intents to consolidate patterns)

## Skill Decisions

The skill determines:
- **Which sub-skills to invoke** based on intent complexity, testing requirements, and project maturity
- **Execution order** to ensure dependencies are satisfied (1.1 before 1.2, all modeling before testing)
- **Required artefacts** to output (SQL models, YAML schemas, documentation, tests)
- **Data contract thresholds** to apply (inherited from `project_standards.md`)

## Output Artefacts

- **SQL:** `models/staging/**/*.sql`, `models/intermediate/**/*.sql`, `models/marts/**/*.sql`
- **YAML:** `models/staging/**/_*.yml`, `models/intermediate/**/_*.yml`, `models/marts/**/schema.yml`
- **Docs:** generated by invoked sub-skills
- **Tests:** generated by 1.4, 2.1, 2.2, 2.3 (DQ checks, unit tests, integration tests, E2E tests)
- **Test Results:** `dbt test` output showing all unit, integration, and E2E tests passing (or failures documented with remediation)

## Implementation Notes

- Always load `project_standards.md` before processing
- Validate input completeness; prompt if missing business intent, metrics, rules, or grain
- Use state-based selectors (`:state:modified+`) for incremental builds
- Document execution log with skill invocation order
- Cross-reference all generated models to entry point intent
- **Generate all models and tests first** (1.1, 1.2, 1.4, 2.1, 2.2, 2.3)
- **Then execute unified `dbt test`** to validate all tests (unit, integration, E2E) pass
- DQ checks (1.4) are generated for reference but NOT included in Phase 2 `dbt test` execution
- If any tests fail, provide clear diagnostic output and suggest remediation steps
- Do not skip or suppress failing tests — report them with root cause analysis
- Document all test results (pass/fail count, coverage summary) as final output
